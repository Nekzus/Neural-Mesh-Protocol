NMP opera bajo un paradigma fundamentalmente distinto a las tradicionales APIs REST o al Model Context Protocol (MCP). Abandona el modelo cliente-servidor JSON-RPC tipo concentrador-y-distribuidor en favor de un **modelo descentralizado de empuje binario (push model)**.

Al orquestar **Lógica en el Origen (Logic-on-Origin)**, NMP elimina las restricciones de gravedad de datos, permitiendo que la lógica del Agente viaje hacia el conjunto de datos en lugar de forzar a que el conjunto de datos viaje hacia el Agente.

## Alcance del Protocolo

NMP comprende varios pilares fundamentales:
- **Especificación NMP**: La implementación estandarizada de mensajes Protobuf, enrutamiento libp2p y restricciones de ejecución binaria WASI.
- **SDKs de NMP**: Bibliotecas oficiales en TypeScript y Rust para integrar sin problemas la malla en las aplicaciones existentes.
- **Motores de Ejecución NMP (Runtimes)**: El motor central (usualmente `wasmtime`) que lleva a cabo el aislamiento y sandboxing matemático.

<Note>
  De forma similar a MCP, NMP es agnóstico respecto al modelo LLM utilizado internamente. Se rige estrictamente por estandarizar *cómo* se intercambian el contexto y la ejecución lógica de forma segura.
</Note>

## Participantes de la Malla

En la Malla Neuronal (Neural Mesh), cada nodo es un Par (Peer), pero asumen roles distintos en base a su propósito computacional:

*   **Sistema Host (Host System)**: El sistema operativo subyacente, base de datos o dispositivo que contiene los datos crudos propietarios.
*   **Nodo de Datos (Servidor)**: Un par NMP especializado que se ejecuta de forma adyacente al Sistema Host. Inicializa el sandbox `WASI` y aguarda la llegada de módulos remotos. Protege al Host de ejecuciones maliciosas.
*   **Nodo Agente (Cliente)**: Típicamente una Inteligencia Artificial Autónoma. Genera bytes de WebAssembly dinámico (`.wasm`) mapeados según sus intenciones y empuja estos bytes a la Malla para encontrar Nodos de Datos correspondientes.

## Capas del Sistema

Para asegurar una latencia ultra-baja y máxima seguridad, NMP está estrictamente dividido en dos capas técnicas, orquestadas a través de tuberías matemáticamente verificables.

### 1. Capa de Transporte (libp2p & QUIC)
La capa de transporte se encarga de descubrir a los pares y de establecer conexiones inquebrantables.
- **Protocolo de Red**: Usa **QUIC**, el cual soporta nativamente la multiplexación sin bloqueos de cabeza de línea (head-of-line blocking) UDP, y **TCP/WebSockets** para compatibilidad de Entornos Navegador (Edge/Browser).
- **Criptografía**: Los handshakes iniciales utilizan el **Noise Protocol** (Ed25519) combinado con **Sobres Autenticados Simétricos (AES-256-GCM)**, profundamente arraigados en la **Criptografía Post-Cuántica (ML-KEM-768 / Kyber)**.
- **Enrutamiento**: Utiliza una **Tabla Hash Distribuida Kademlia (DHT)** altamente optimizada. Los nodos agente no necesitan direcciones IP grabadas en código duro ni DNS centralizado; simplemente le preguntan a la Malla "¿Quién puede cumplir con esta solicitud?", y Kademlia enruta el paquete binario de manera autónoma.

### 2. Capa de Datos (Protobuf & Tonic)
Una vez la capa de transporte conecta el Agente con el nodo de Datos, la Capa de Datos dicta la estructura de la solicitud de ejecución.
- **Codificación**: Abandona sus representaciones JSON voluminosas. Todos los mensajes se empacan estrechamente en un Protocolo Binario de Búferes (`.proto`).
- **Framework RPC**: Usa **Tonic (gRPC)** para una transmisión bidireccional ágil como el relámpago.
- **Estructura del Paquete (Payload)**: La Capa de Datos define los elementos primitivos `LogicRequest` y `LogicResponse`, transmitiendo el WASM AST, los identificadores de capacidad y las pruebas criptográficas (`ZK-Receipts`).

### 3. Capa de Verificación y Aislamiento (Ejecución)
La frontera final antes de tocar el metal.
- **Validación AST del Guardián**: Rastrea e intercepta vulnerabilidades de Host-Escape sobre importaciones WebAssembly antes de iniciar el Sandbox.
- **Enclaves de Hardware (TEEs)**: Opcionalmente, arranca todo el motor de ejecución de `wasmtime` dentro de un enclave de hardware propietario como un Enclave Intel SGX o AWS Nitro, para un blindaje de la RAM evitando inspectores del sistema administrativo remoto.
- **Evidencia Lógico-Matemática Zero-Knowledge**: Opcionalmente, empodera los zkVMs (como `RISC Zero`) para que el servidor devuelva deducciones operadas selladas matemáticamente de que nada de lo generado fue falseable ni mutado.

Aquí está el ciclo vital exacto de una ejecución NMP, que refleja una paridad 1:1 entre nuestros Modelos Arquitectónicos, el Backend de Rust y el SDK para Node.js:

### 1. El Cliente Emisor (`TypeScript Client`)
- **Compilación WebAssembly:** La lógica dinámica del LLM se evalúa y transforma a sí misma de inmediato en un stream binario de formato `.wasm`.
- **Sensor Guardián de Tipo Árbol AST para Clientes (`GuardianTS`):** Antes de toda la subida hacia la red, la inspección heurística final de pre-vuelo anula bloqueos y alcances ilegales, y corta la emisión para prevenir infecciones.
- **Post-Quantum Key Exchange:** El SDK genera un secreto cuántico `Kyber768`, destruyendo por pura matemática la agresión conocida de robar el protocolo y analizarlo con superordenadores *Harvest Now, Decrypt Later*.
- **Cifrado del Paquete (Payload Encryption):** El manifiesto `.wasm` es precintado de modo cerrado en formato criptográfico simétrico bajo `AES-256-GCM` valiendo los nonces aleatorios.

### 2. El Tránsito por Kademlia (`Zero-Trust Mesh`)
- **Libp2p Transport:** Paquetes inyectados sobrepasan distancias viajando por sustratos invisibles Malla empleando latencia mínima `TCP/QUIC`.
- **Multiplexed Secure Streams:** Mientras el gestor `Yamux` invoca sus recursos manejando miles de bifurcaciones P2P asincrónicas a la vez, el tránsito fluye resguardado vía *handshakes* robustos de Noise P2P.

### 3. La Ingesta Nivel Host Origen (`Rust Server`)
- **Terminator de Transporte:** Al impactar su punto central, este de-oculta todo su ecosistema interno para destrabar Kyber Secrets resolviendo localmente un pre-vuelo inverso en pura decodificación. Extrae su núcleo.
- **AST Guardian (Sensor Centinela de tiempo Cero):** El parseador asíncrono sobre el analizador de Rust (`wasmparser`) fracciona lo inyectado AST a velocidades récord, fulminando instantáneamente cualquier llamado host fraudulento detectado inyectado o subido.
- **Wasmtime / Limites de Arena Sandboxing:** El gigante subyacente y base fundacional de Bytecode Alliance enciende este entorno microscópico blindado en Memoria RAM con márgenes constrictivos hostiles para que un loop del tamaño petabyte de Data local fluya impermutable bajo absoluto silencio remoto pero bajo latencia red de nivel asincrónico 0.

### 4. Mathematical Return Cero-conocimiento (`ZK-Receipt`)
- **Evidencia Cero Riesgo Matemático:** Opcionalmente, una Máquina Virtual de Cero Conocimiento como `RISC Zero` forja internamente sellos de validación (ZK-Receipt) e indescifrables garantizando lo computado de forma criptográfica sin compartir la data subyacente de donde derivan.
- **Stream Binario Yamux**: El resultado es retornado hacia su emisor en la tubería múltiple y compacta multiplex gRPC para empoderar al Agente IA Autónomo en todo momento devolviendo respuestas infalibles en certidumbre matemática pura.

Esta arquitectura End-to-End provee no sólo la más colosal velocidad requerida y teóricamente insuperable para IAs Distribuidas Autónomas del momento sino una base inviolable grado Privacy Zero-Trust sobre sistemas puramente y matemáticamente verificables para todo el ciclo de transacciones.
